# Relay (주간보고 정리 agent)

운영 중인 시스템의 **주간 업무 보고서**를, "지난주에 한 일의 단순 나열"이 아니라
**①안정적으로 운영되고 있다는 증거 + ②향후 리스크 관리**에 초점을 맞춰 작성·정리하는 도구.

이름 *Relay*는 핵심 동작에서 왔다 — 지난주의 미완료 task를 이번 주로 **바통처럼 넘기고**(carry-over),
같은 작업을 주차를 가로질러 한 줄기로 잇는다.

보고서는 세 축으로 구성된다: **정량 지표(숫자) / 정성 활동(한 일) / 예측(다음 주·리스크)**.

## 핵심 기능

- **전주 기반 초안 자동 생성** — 지난주 미완료 작업 이월 + "다음주 계획" 항목을 금주 신규 업무로 승격
- **신규 업무·이슈·지표 입력** 및 요약·정리
- **연관 과거 내용 정리** — 재발성 이슈 등 의미적으로 연관된 과거 항목 연결(RAG, 단계적 도입)
- **월간·연간 보고 자료 생성** — 누적 데이터 집계
- **사용자 편집 가능한 템플릿** — 보고서 카테고리·지표를 YAML로 커스터마이즈

## 설계 원칙 (요약)

- **숫자는 코드, 서술은 LLM** — 집계·추세는 코드가 계산하고, LLM은 서술·맥락화만 (환각 방지)
- **구조화 DB가 원본, 벡터(RAG)는 파생 인덱스** — 이월·집계는 정확 조회, 연관 정리만 의미 검색
- **무거운 LLM 프레임워크 지양** — 가벼운 부품(anthropic SDK · Pydantic · Jinja2 · sqlite-vec) 조합

> 전체 설계 결정·작성 워크플로우·검증 단계는 [`CLAUDE.md`](./CLAUDE.md) 참조.

## 개발 환경

- Python **3.12** (uv로 관리)
- 패키지/가상환경: [uv](https://docs.astral.sh/uv/)

```bash
uv sync                 # 가상환경(3.12) 생성 + 의존성 설치
uv run relay version    # CLI 동작 확인
uv run pytest           # 테스트
uv run ruff check .     # 린트
```

RAG 기능(선택)을 쓰려면:

```bash
uv sync --extra rag
```

## 사용해보기 (Quickstart)

`relay` 를 실행하면 **대화형 slash 쉘**이 뜬다. 명령을 연속으로 입력하며 활성 컨텍스트(주/시스템)는
세션 동안 유지된다. 현재 구현된 **task 관리 루프**(추가·메모·상태·이력)를 바로 돌려볼 수 있다
(LLM·이월 자동 초안은 아직 없다 — 아래 "상태" 참조).

```bash
# (권장) 실제 홈을 더럽히지 않게 데이터를 임시 위치로 보낸다
export RELAY_HOME=/tmp/relay_try
export RELAY_DB=/tmp/relay_try/relay.db

uv run relay        # 대화형 쉘 시작
```

쉘 안에서 (맨 앞 `/` 는 생략 가능). **slash 없는 일반 문장은 LLM 이 카테고리를 분류해 등록**한다:

```text
[2026-W26 / 기본] > 어제 첨부 다운로드가 자꾸 느려서 스토리지 로그 봤어
  🤖 분류: 장애·이슈 대응 (incident)
     제목: 첨부 다운로드 간헐 지연 — 스토리지 로그 점검
     [y=등록 / n=취소 / 새 제목 입력=그 제목으로 등록]
[2026-W26 / 기본] > y
  ✓ 등록됨  [1] ...

[2026-W26 / 기본] > /add 장애 "첨부 다운로드 간헐 지연"   # 카테고리 직접 지정도 가능
  ✓ 등록됨  [2] 첨부 다운로드 간헐 지연  (장애·이슈 대응)
[2026-W26 / 기본] > /add 정기 보안 패치 적용
  ✓ 등록됨  [2] 보안 패치 적용  (정기 작업)
[2026-W26 / 기본] > /note 1 I/O 로그 수집 완료
[2026-W26 / 기본] > /update 2 완료
[2026-W26 / 기본] > /list
  [1] (진행중) 첨부 다운로드 간헐 지연
  [2] (완료) 보안 패치 적용
[2026-W26 / 기본] > /history 1
[2026-W26 / 기본] > /use 포털        # 활성 시스템 전환
[2026-W26 / 포털] > /week 2026-W25   # 활성 주차 전환
[2026-W26 / 포털] > /help
[2026-W26 / 포털] > /quit
```

동작 메모:

- **자연어 입력**: slash 없이 문장을 입력하면 LLM 이 카테고리 분류 + 제목 요약을 제안하고, 확인 후 등록한다. API 사용 시 입력 메모가 외부로 전송됨에 유의.
- **API 키·모델 관리**: `~/.config/relay/config.ini`(`[llm]` 섹션, `RELAY_CONFIG` 로 경로 변경 가능)에 저장한다. 키가 없으면 쉘 첫 실행 시 한 번 입력받아 저장(권한 600)하고, 입력을 비우면 오프라인 규칙 분류로 진행한다. `ANTHROPIC_API_KEY` 환경변수가 있으면 그쪽이 우선한다.
- **slash 명령**: `/add`·`/list`·`/update`·`/note`·`/history`·`/use`·`/week`·`/help`·`/quit`. `/task add` 표기도 허용. 번호(1,2,3…)는 `/list` 에 보이는 값.
- **활성 컨텍스트**: 시작 시 이번 업무주(KST 월~금) / 마지막 사용 시스템. 세션 중 `/use`·`/week` 로 전환.
- **카테고리**: 키(`incident`) 또는 이름 일부(`장애`)로 지정. 모호하거나 없으면 가능한 값을 안내한다.
- **데이터 위치**: 기본은 `~/.config/relay/relay.db`. 템플릿 사본은 `~/.config/relay/template.yaml`(첫 실행 시 자동 생성).
- 실험 후 정리: `rm -rf /tmp/relay_try`.

## 프로젝트 구조

```
src/relay/              애플리케이션 코드 (src 레이아웃)
  cli.py                단발 CLI 진입점 (얇게 — 서비스 레이어 호출만)
templates/              사용자 편집 가능한 보고서 템플릿 (YAML)
examples/               렌더링 결과 예시 (golden output)
```

## 상태

초기 개발 단계(MVP 마일스톤 1 진행 중). 상세 작업 목록은 [`task.md`](./task.md), 설계는 `CLAUDE.md` 참조.

- **구현됨**: week 변환 유틸 · 템플릿 로더/검증 · SQLite 스키마+Task 모델 · 대화형 slash 쉘(`/add`·`/list`·`/update`·`/note`·`/history`·`/use`·`/week`)(상태/이월 분리·`thread_id`·진행 메모) · LLM provider 어댑터 + **자연어 캐처**(문장 → 카테고리 분류·제목 요약, 키 없으면 오프라인 폴백).
- **다음**: `/draft`(미완료 이월 + 다음주계획 승격) → 보고서 Markdown 렌더 → LLM 서술 연동 → 집계 보고 → RAG.
