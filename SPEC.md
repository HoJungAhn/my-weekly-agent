프로젝트를 구상하고 있어 기능은 아래와 같아 
- 제목 : 주간보고 정리 agent
  - 현재 운영중인 시스템에 대한 weekly 보고서를 매주 작성하는데 보고서는 "지난주에 뭘 했는지"의 단순 나열보다,시스템이 안정적으로 돌아가고 있다는 증거 + 앞으로의 리스크 관리를 위한 보고서를 작성하고 있어
  - 그리고 운영에 대한 내용이외에 예방 및 개선활동을 작성하며 아래와 같은 구성으로 작성
    - 정량 지표(숫자) 
    - 정성 활동(한 일)
    - 예측(다음 주/리스크)
  - 주간업무 보고는 다음과 같은 템플릿으로 구성
    - 시스템 운영 현황
      - 가동률/안정성, 주요 지표(트래픽, 처리건수, 응답시간), 정상 운영 여부
    - 장애·이슈 대응
      - 발생 장애 내역, 원인, 조치 결과, 재발 방지책
    - 정기 작업
      - 배포, 패치, 백업, 모니터링, 정기 점검 결과 
    - 개선·요청 처리 
      - 사용자 요청(ITS) 처리, 기능 개선, 데이터 정정 
    - 다음 주 계획 / 리스크
      - 예정 작업, 점검 필요 사항, 잠재 위험
  - 금주 주간업무 작성을 하기 위해서는 지난주 미완료 작업내용과 금주 새로운 내용을 작성
- 주간보고 서비스 기능
  - 금주 주간업무를 작성하기 위해 이전에 저장된 주간업무내용을 참조해서 금주 초안을 자동 생성
  - 금주 신규 업무에 대해 입력을 받고 요약 및 정리
  - 매주 작성된 주간업무 내용을 기반으로 월간보고 및 연간보고 자료 생성 
- 개발기술 내용
  - python 기반 RAG를 이용하여 주간업무내용을 저장 및 LLM서비스를 연동하여 주간업무 내용을 정리
  - openclaw 와 같은 인터페이스 방식으로 구성하고 /(slash) 명령을 이용하여 task 추가/변경, summery, 작성된 weekly 수정 등 기능제공
이 기능을 만들기 위해 조언을 해줘
